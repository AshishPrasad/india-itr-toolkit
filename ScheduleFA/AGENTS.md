# AGENTS.md

Guidance for AI agents and contributors working in this repository.

## Repository purpose

This folder contains **Schedule FA (Foreign Assets)** helpers for Indian ITR —
two command-line tools:

- **`dividend-contribution`** — computes per-lot dividend contribution from a
  dividend schedule and a stock-transaction ledger (buys and sells, including
  partial sells).
- **`schedule-fa-csv`** — generates the CSV for **section A3 of Schedule FA**
  (Table A3 — financial interest in any entity) from a per-lot Excel summary.

See `README.md` for full usage.

> **Privacy:** Any real workbook in this directory may contain personal
> financial data. Do **not** copy real values (amounts, holdings, totals,
> account details) into source code, tests, documentation, commit messages, or
> logs. Tests must use the synthetic sample data generated under
> `tests/.tmp_data/` only.

## Project structure conventions

When adding tools or features, keep the repo-style layout:

- **Modular code under `src/`** — one responsibility per module. Each tool is
  its own package (e.g. `dividend_contribution`, `schedule_fa_csv`); shared
  Excel-parsing helpers (`parsing`, `headers`) live in `dividend_contribution`
  and are reused. Keep core logic pure (no file/Excel I/O) so it is easy to unit
  test; isolate I/O in `reader(s).py` / `writer.py`.
- **Tests under `tests/`** — use `pytest`, cover each module, and include
  end-to-end tests. Provide **sample Excel sheets** and **validation** of
  results (totals must foot). Sample data is generated by
  `tests/generate_sample_data.py` into the gitignored `tests/.tmp_data/`
  folder (generated artifacts are not committed).
- **A `README.md`** documenting purpose, input formats, usage, and how to run
  tests.

## Build, run, and test

```bash
pip install -e .[dev]          # install package + dev deps (pytest)
python -m pytest               # run the full test suite (from ScheduleFA/)
python -m dividend_contribution --help
python -m schedule_fa_csv --help
```

The package uses a `src/` layout; `pyproject.toml` sets
`tool.pytest.ini_options.pythonpath = ["src"]` so tests run without an install.
Run `python -m pytest` from the `ScheduleFA/` directory; a session-scoped fixture
regenerates synthetic sample data into `tests/.tmp_data/` first. On Windows,
`dividend-contribution.cmd` and `schedule-fa-csv.cmd` wrap the respective
`python -m <package>` invocations.

## Coding conventions

- Python 3.9+; standard library + `openpyxl` only for runtime.
- Header detection is tolerant: exact column-name match first, then aliases;
  always allow explicit overrides via CLI flags.
- Parse currency/percent text (e.g. a `₹`-prefixed value) defensively; treat
  blanks/`-`/`n/a` as missing.
- Allocation must reconcile: per-lot contributions sum to the allocated total
  (use largest-remainder rounding to keep the output column footing exactly).
- Add or update tests for any behavior change; keep tests hermetic (synthetic
  data, `tmp_path` for outputs).
